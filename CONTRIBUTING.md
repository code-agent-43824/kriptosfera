# Contributing to Kriptosfera

This document covers local development of the Go launcher. The Windows-only
build/packaging flow (payload, Chromium runtime, CryptoPro bundles) is described
in [`README.md`](README.md) and the documents under [`docs/`](docs/README.md).

## Prerequisites

- Go (see the `go` directive in [`go.mod`](go.mod); CI uses 1.24).
- For full Windows builds: PowerShell 7 (`pwsh`) and the scripts in `build/`.

## Repository layout

```text
cmd/kriptosfera-launcher/   entrypoint (main)
internal/bootstrap/         payload preparation + runtime launch
internal/config/            runtime config (build-time) + app config (payload)
internal/logging/           append-only launcher log
payload-template/           payload skeleton (config, diagnostics, extension)
build/                      PowerShell build/packaging scripts
tools/windows/              read-only Windows diagnostics scripts
docs/                       concept, plans, and architecture notes
```

See [`docs/architecture.md`](docs/architecture.md) for the launcher flow and the
runtime AppData layout.

## Building and testing on any platform

Real `payload.zip` / `cryptopro-plugin.zip` artifacts are generated by the build
scripts and are not committed. The repository ships **zero-byte placeholders**
at `internal/bootstrap/payload.zip` and `internal/bootstrap/cryptopro-plugin.zip`
so the `go:embed` directives are satisfied and the launcher compiles on a clean
checkout. An empty embed is treated as "bundle not embedded"; Windows builds
overwrite the placeholders with the real artifacts.

Standard checks (run from the repository root):

```sh
gofmt -l .                              # formatting (no output = clean)
go vet ./...
go test ./...
GOOS=windows GOARCH=amd64 go build ./...               # embedded launcher
GOOS=windows GOARCH=amd64 go build -tags remote ./...  # thin/remote launcher
```

On non-Windows hosts the launcher performs a diagnostics "dry-run" instead of
starting Chromium, so `go test ./...` exercises the full bootstrap logic without
a browser.

## Full Windows build

```powershell
./build/build-payload.ps1
./build/build-launcher.ps1 -PayloadMode embedded -PayloadZip dist/payload.zip -PayloadMetadata dist/payload.json
./build/build-launcher.ps1 -PayloadMode remote -UsePayloadLock -PayloadLockPath build/payload-lock.json
```

These are also run by the `build-windows` GitHub Actions workflow.

## Conventions

- Commits follow [Conventional Commits](https://www.conventionalcommits.org/)
  (`feat:`, `fix:`, `docs:`, `chore:`, `ci:`), matching the existing history.
- Exported Go symbols should carry godoc comments.
- Keep all CryptoPro / Chromium binaries out of Git; fetch or generate them via
  the build scripts and pin them with SHA-256/size lock files.
- Update [`CHANGELOG.md`](CHANGELOG.md) for user-visible changes.
